[首页][blog_link] # 为什么要用python 的虚拟环境?(virtualenv) 因为pip 直接安装 即 pip install xxxx 是安装到Global 全局的环境之中。 而虚拟环境可以将包（模块）安装在文件夹内,便于管理包的版本 # 前置需求 ： ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ pip install virtualenv ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ # 设置python虚拟环境 - python 的虚拟环境 本质是一个文件夹 So：删除直接 rm -rf 即可 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ windwos :python3 -m venv path\venv linux : virtualenv path/venv ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ # 进入虚拟环境 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ windows：.\venv\Scripts\activate.ps1 linux : source ./venv/bin/activate ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ PS:进入后虚拟环境后,终端前会有 “（venv)”的字样 # 安装所需要的外部包 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ pip install -r requirements.txt ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ requirements.txt 内容： - colorma - xxxxxx - xxxxx - xxxxx PS:一键安装所需所有包，省事 # 虚拟环境配置完成 [blog_link]:https://xtccc.github.io/small_parking.github.io/
